# Post-v1.5 improvements — queue

> Status: **Planning** · 2026-06-14. A queue of UX / polish / feature work agreed after the v1.5.0
> device-testing pass. Each item is a stub here; the larger ones get their own dated plan doc when
> picked up. All items keep the no-`INTERNET` invariant.

## Queued

1. **Share to Stela (receive shared text)** — ✅ **Done (v1.6.0, unreleased).** The app is a
   `text/plain` share target; sharing text opens a new note prefilled with it. Offline, no new
   permission. See [2026-06-14-share-to-stela.md](2026-06-14-share-to-stela.md).

2. **First-run onboarding** — ✅ **Done (v1.6.0, unreleased).** A skippable 3-pane first-launch flow
   (concept → notifications → keep-alive) that explains the concept, requests `POST_NOTIFICATIONS` with
   context, and nudges battery-optimisation exemption. Replayable via Settings → About. See
   [2026-06-14-first-run-onboarding.md](2026-06-14-first-run-onboarding.md).

3. **Editor discard-changes confirm** — ✅ **Done (v1.6.0, unreleased).** Back in the full editor shows a
   "Discard changes?" confirm (Discard / Keep editing) when the note is dirty; Save stays enabled but is
   a no-op that doesn't bump Modified Time when nothing changed. See
   [2026-06-14-editor-discard-confirm.md](2026-06-14-editor-discard-confirm.md).

4. **Scheduled-note list indicator** — a note with a `pinAt`/`unpinAt` schedule looks identical to
   any other in the list (`NoteRow`). Add a small schedule/clock badge so scheduled notes are
   discoverable at a glance. *Stub — to plan.*

5. **Material You / dynamic colour** — `StelaTheme` uses fixed brand colours only. Offer an optional
   "Use system colours" mode using `dynamicLightColorScheme` / `dynamicDarkColorScheme` (Android 12+),
   falling back to the brand palette below 12. *Stub — to plan.*

6. **Haptic feedback on pin toggle** — a light haptic when toggling a note's pin (list row and editor)
   so pinning feels tactile. *Stub — to plan.*

7. **Duplicate note action** — duplicate an existing note (fresh id, unpinned, copied
   title/description/emoji). *Stub — to plan.*

## Deferred / not now (recorded for context)

- **Checklist / to-do notes** — out of scope for this project; kept in mind as a possible future
  headline feature.
- **Richer empty state** — current instruction text is considered sufficient.
- **Notification Snooze action** — Snooze stays editor-only for now (this was previously shelved).
- **Swipe actions on list rows** (swipe-to-archive / delete) — good idea, deferred.
